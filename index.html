What is a Club Penguin Remake?

Club Penguin is a dead virtual world in which children could create accounts as penguins and roam the island to make friends. It was previously owned by Disney. In March of 2017, Club Penguin was shut-down. A Club Penguin remake is essentially a re-run of the original game, Club Penguin, with the same visual files as the original game yet different back-end code. Essentially, the creator of one of these would have to host all of the original visual files from Club Penguin on their own domain and would have to re-write the server code after heavy debugging. Then, the creator of this game would publish this content to the world on a Virtual Private Server (VPS) and make it public.

How did Vintage Penguin come to be?

From around 3rd grade, I've always had an obsession with modifying the original game, Club Penguin. I would test scripts to cheat-in coins or just change the speed of the game for my pure entertainment. I spent many hours just looking at the code because it excited me to see what would happen. Eventually, the game died in March of 2017. Club Penguin Remakes were a "trend" before Club Penguin died; however, I wanted to host my own at this point because I felt ready in my process of development and also had a pure desire to push my novel ideas onto the game. So, I met with a couple peers and began to code.

Code Behind the Scenes

Truly speaking, there were a lot of elements that were pushed to create this game. This section will briefly highlight them.

Backbone

With just the visual files, penguins cannot login into the game. The game functions by listening to two constant server connections to let any penguins log into the game.



To code this part of the game, I had to process the "receiving" output from the visual files of a penguin logging in and verify their login information by using a random number algorithm. Direct passwords would not be sent for security reasons. After successfully processing the user's login information, a connection to a local SQL database would be made to get the user's items, igloo, ban status, friend's list, and other stuff from the database. Here's an example of a handler I coded for igloo competitions, written in PHP.



This information would be communicated using packets that essentially detail information about the user; in this example, a loadPlayer (lp) packet is sent to give information about the user, "test9gdfgd" and his clothes. After getting the user to load in the world, the rest of the backbone became easy. The world of Vintage Penguin functioned off of room packets. Essentially, to show another the actions of the main user, the packets being sent by the main user would be repeated to all the penguins in the room. Essentially, each penguin had a socket connection to the game so they would receive any packets being sent. So, to make the world of Vintage Penguin function, all I had to do was code response methods for any user actions. For example, when a user threw a snowball, I had to simply tell the game to send this action to all users in the room, and the visual files would handle the rest. This pulls together actions from changing rooms to saying "Hello" to customizing one's igloo.

Front-end

As there were other Club Penguin Remakes out there with different code, my server functioned around a theme to attract users. The theme of my server was the years of Club Penguin from 2005-2009. My intended audience was veteran Club Penguin players that wanted to satisfy their nostalgia. To do this, I had to re-texture all of the media-files from Club Penguin as well as modify the code to the looks of 2006. To accomplish this, I pulled together many old .swf (Adobe Flash) files and stole assets to create one giant file. Then, I added the handlers that connected with the back-end so that the user would still login with these re-designs. Security

While I coded this, one of my biggest concerns was security. To make sure no exploits were used, I would always stay current on any news concerning the frameworks I utilized. Additionally, I made sure to never directly send passwords throughout connections and also encrypt them using BCrypt (link me!) to make the passwords as secure as possible. Overall, nothing significant happened to my remake. We also did not have any conflicts with Disney via 17 U.S. Code ยง 107 on Fair use (link me!).
